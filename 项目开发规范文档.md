## 1. 项目概述

本项目是一个完整的外卖系统，包含秒杀功能，具体包括：
- **后端服务**：Spring Boot 2.7.3 + MyBatis + MySQL + Redis
- **管理后台**：Vue 2.6.10 + TypeScript + Element UI
- **用户端**：UniApp 微信小程序
- **秒杀功能**：高并发秒杀系统，支持Redis缓存、Lua脚本、库存管理等

// utils/request.js
export function request({url='', params={}, method='GET'}) {
  const header = {
    'Accept': 'application/json',
    'Content-Type': 'application/json',
    'authentication': store.state.token
  };
  
  return new Promise((resolve, reject) => {
    uni.request({
      url: baseUrl + url,
      data: params,
      header: header,
      method: method,
      success: (res) => {
        if (res.data.code == 200 || res.data.code === 1) {
          resolve(res.data);
        } else {
          reject(res.data);
        }
      },
      fail: (err) => {
        reject(err);
      }
    });
  });
}

## 5. 数据库设计规范

- 表名具有描述性，如：`address_book`、`order_detail`

- 字段名使用小写字母，单词间用下划线分隔
- 主键统一命名为`id`
- 外键以`_id`结尾，如：`user_id`、`category_id`、`activity_id`、`seckill_goods_id`
- 时间字段以`_time`结尾，如：`create_time`、`update_time`、`start_time`、`end_time`
- 状态字段使用`status`
- 金额字段使用`DECIMAL(10,2)`类型
- 数量字段使用`count`或`quantity`
- 版本字段使用`version`（用于乐观锁）
- Redis相关字段以`redis_`开头

## 8. 秒杀功能开发规范

### 8.1 秒杀业务特点
- **高并发**：需要处理大量用户同时访问
- **库存管理**：防止超卖，确保库存准确性
- **限购控制**：用户购买数量限制
- **时间控制**：严格的活动时间管理
- **性能要求**：响应时间要求极高

### 8.2 技术架构规范

#### 8.2.1 缓存策略
- **Redis预热**：活动开始前将商品信息预加载到Redis
- **库存管理**：使用Redis管理实时库存，避免数据库压力
- **用户限购**：Redis记录用户购买记录，实现限购控制
- **活动状态**：Redis缓存活动状态，减少数据库查询

#### 8.2.2 防超卖机制
```java
// 使用Lua脚本保证原子性
@Service
public class SeckillStockService {
    
    @Autowired
    private RedisTemplate redisTemplate;
    
    public boolean deductStock(Long goodsId, Integer quantity, Long userId, Integer limitCount) {
        // Lua脚本保证库存扣减的原子性
        String script = "local stock = redis.call('GET', KEYS[1])\n" +
                       "local userPurchased = redis.call('GET', KEYS[2]) or '0'\n" +
                       "if not stock or tonumber(stock) < tonumber(ARGV[1]) then\n" +
                       "    return {code = 50006, msg = '库存不足'}\n" +
                       "end\n" +
                       "if tonumber(userPurchased) + tonumber(ARGV[1]) > tonumber(ARGV[2]) then\n" +
                       "    return {code = 50008, msg = '用户已达购买上限'}\n" +
                       "end\n" +
                       "redis.call('DECRBY', KEYS[1], ARGV[1])\n" +
                       "redis.call('INCRBY', KEYS[2], ARGV[1])\n" +
                       "return {code = 1, msg = 'success'}";
        
        // 执行Lua脚本
        return executeScript(script, goodsId, quantity, userId, limitCount);
    }
}
```

#### 8.2.3 订单处理流程
1. **资格检查**：验证用户购买资格
2. **库存扣减**：使用Lua脚本原子性扣减库存
3. **订单创建**：创建秒杀订单记录
4. **支付处理**：处理订单支付
5. **库存同步**：定时同步Redis库存到数据库

## 11. 安全规范

### 11.1 认证授权
- 使用JWT进行用户认证
- 管理端和用户端使用不同的密钥
- Token有效期设置为2小时

### 11.2 数据安全
- 密码使用MD5加密存储
- 敏感数据传输使用HTTPS
- SQL注入防护使用MyBatis参数化查询

### 11.3 接口安全
- 所有接口都需要登录认证（除登录接口外）
- 使用拦截器进行权限校验
- 敏感操作记录日志
