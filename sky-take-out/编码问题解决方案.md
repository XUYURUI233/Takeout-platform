# 编码问题解决方案

## 问题描述
启动应用时出现以下错误：
```
Caused by: org.xml.sax.SAXParseException: 2 字节的 UTF-8 序列的字节 2 无效。
```

## 问题原因
XML映射文件中的中文注释或内容存在编码问题，导致MyBatis解析失败。

## 解决方案

### 1. 立即解决方案
已修复 `SeckillGoodsMapper.xml` 文件的编码问题，移除了可能导致编码冲突的中文注释。

### 2. 预防措施

#### 2.1 IDE设置
确保IDE使用UTF-8编码：
- **IntelliJ IDEA**: File → Settings → Editor → File Encodings
  - Global Encoding: UTF-8
  - Project Encoding: UTF-8
  - Properties Files: UTF-8

#### 2.2 Maven配置
在 `pom.xml` 中确保编码配置：
```xml
<properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
</properties>
```

#### 2.3 文件编码检查
所有XML文件都应该：
1. 以 `<?xml version="1.0" encoding="UTF-8" ?>` 开头
2. 避免在XML中使用中文注释，改用英文注释
3. 如需中文，确保文件保存为UTF-8格式

### 3. 修复后的文件列表
- ? `SeckillGoodsMapper.xml` - 已修复
- ? `SeckillStockLogMapper.xml` - 正常
- ? `SeckillUserRecordMapper.xml` - 正常
- ? `SeckillActivityMapper.xml` - 正常

### 4. 验证步骤
1. 清理编译缓存：`mvn clean`
2. 重新编译：`mvn compile`
3. 启动应用：`mvn spring-boot:run`

### 5. 后续注意事项
1. 新建XML文件时，统一使用英文注释
2. 如必须使用中文，确保文件编码为UTF-8
3. 定期检查文件编码一致性
